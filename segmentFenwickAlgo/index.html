Segment Tree

Range queries + point/range updates in O(log N). Stored as an array of size 4N. The most powerful general-purpose range structure in competitive programming.

Core Template — Build, Query, Update

Master the 3 operations. Node i has children 2i and 2i+1. Build O(N), query O(log N), update O(log N). Write this from memory before moving forward.

4N arrayO(N) buildO(log N) query/updateSum · Min · Max
4 problems · ~50 min
💡 Concept

Build bottom-up. Query by splitting into O(log N) disjoint ranges. Update by walking root → leaf, rebuilding each ancestor. The merge function (sum/min/max/GCD) is the only thing that changes between problems. Everything else stays identical.

Lazy Propagation — Range Updates in O(log N)

One rule: pushdown before accessing children. Each node gets a lazy tag. When visiting a node, flush the tag to children first. This single idea unlocks 80% of hard seg tree problems.

Lazy tagPushdown patternRange add / assign
4 problems · ~55 min
💡 Concept

When updating range [l,r], mark parent nodes with a pending lazy tag. When you later need to query a child, call pushdown(node) first to push the tag down. Both update and query remain O(log N). For multiple lazy types (add + multiply), define how they compose before coding.

Fenwick Tree (BIT)

Simpler than seg tree, same O(log N). Uses the lowbit trick: i & (−i). Ideal for prefix sum + point update. 10-line template, unbeatable constant factor.

13
problems
B1
🟢 Beginner ⏱ Hour 4 — last 30 min

The Lowbit Trick — Core Template

update propagates by i += i&(−i). query traverses by i −= i&(−i). Range [l,r] = query(r) − query(l−1). Memorize these 10 lines completely.

i & (-i)O(log N) update+queryHalf the code of seg tree
4 problems · ~35 min
💡 Concept

BIT[i] stores the sum of the last i&(−i) elements. BIT[6] = binary 110, last bit = 2, so BIT[6] covers 2 elements. update adds to i then jumps to parent via i += i&(−i). query sums prefixes by stripping the last set bit. The beauty: this traversal is naturally O(log N).

Value-Indexed BIT + Coordinate Compression

Compress values into [1..N]. BIT indexed by value. query(x) = "how many elements ≤ x seen so far". Solves most Hard counting problems in one pattern.

Right-to-left sweepValue-indexed BITCounting pairs
4 problems · ~55 min
💡 Concept

Master pattern: process elements right-to-left. For each a[i], query BIT for count in [1, a[i]−1] = smaller elements to its right. Then insert a[i]. This single loop solves inversions, reverse pairs, and "count smaller" all in O(N log N). LC #315 and #493 are the canonical interview versions of this pattern.

Range Updates, 2D BIT + Expert Patterns

Range add using two BITs (dual BIT). 2D Fenwick for matrix queries. BIT walking for k-th element in O(log N). These are the techniques that separate good from great.

Dual BIT2D FenwickBIT walking O(log N)
5 problems · ~50 min
💡 Concept

BIT walking for k-th element in O(log N): start pos = 0. For each bit from high to low — if BIT[pos+bit] < k, then pos += bit and k -= BIT[pos]. After the loop, pos+1 is the answer. Beats binary-search + query (O(log²N)) and is the cleanest advanced BIT trick.
